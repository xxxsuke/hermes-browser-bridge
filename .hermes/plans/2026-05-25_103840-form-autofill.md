# 表单自动填充 (Form Auto-Fill) 功能实现计划

**日期:** 2026-05-25
**项目:** Hermes Browser Bridge
**目标:** 为项目增加完整的表单自动检测 + 智能填充功能

---

## 一、现状分析

### 现有能力
- **`content.js`**: 有一个非常基础的 `fillForm` 函数，接受 `[{selector, value}]` 数组，逐个调用 `writeText` 填充
- **`background.js`**: **`fill_form` 不在 contentOps 列表中**，该命令实际上不可用！
- **`hermes_client.py`**: **完全没有 `fill_form` 相关命令**
- **已有 `writeText`**: 支持原生 setter 绕过 Vue/React 劫持（好的基础）

### 需要补齐的能力
1. 自动检测页面上所有表单字段（不依赖用户提供 selector）
2. 智能匹配字段类型（根据 label、placeholder、name、id、role、aria-label）
3. 支持多种表单控件（text、select、checkbox、radio、date、file、textarea）
4. 支持填充模板（用户预设数据，自动匹配）
5. 端到端 CLI 命令（从 Hermes 侧调用）
6. 表单提交验证（确认字段填对了）

---

## 二、功能架构

```
Hermes Agent / CLI
    │  fill_form / detect_fields / auto_fill
    ▼
background.js (消息路由)
    │  contentOps 新增 fill_form, detect_fields, auto_fill
    ▼
content.js (页面操作)
    ├─ detectFields()     → 扫描表单结构，返回字段元数据
    ├─ fillForm(params)   → 按 selector 数组填充（现有升级版）
    └─ autoFill(params)   → 智能填充（自动匹配字段）
```

## 三、详细步骤

### 步骤 1: `extension/content.js` — 表单检测引擎 ✅ [核心]

新增函数:

#### 1.1 `detectFields(params)` — 自动检测表单字段

```
功能: 扫描页面上的 <form> 元素或特定容器内的所有表单控件
返回: 完整的表单字段元数据

输入参数:
  - formSelector (可选): 限定检测某个 form
  - includeHidden (默认 false): 是否包含 hidden 字段

返回结构:
{
  "forms": [
    {
      "selector": "form#login",
      "id": "login",
      "action": "/login",
      "method": "post",
      "fields": [
        {
          "selector": "#email",      // 最佳 CSS 选择器
          "type": "email",           // 字段类型
          "name": "email",           // name 属性
          "id": "email",             // id 属性
          "label": "邮箱地址",        // 关联的 label 文本
          "placeholder": "请输入邮箱",
          "required": true,          // 是否必填
          "autocomplete": "email",   // autocomplete 提示
          "tagName": "input",        // 标签名
          "value": "",               // 当前值
          "readonly": false,         // 是否只读
          "disabled": false,         // 是否禁用
          "options": [               // select/radio 的选项
            { "value": "male", "text": "男", "selected": false }
          ],
          "rect": { "x": 0, "y": 0, "w": 200, "h": 30 }  // 位置
        }
      ]
    }
  ],
  "totalFields": 5,
  "totalForms": 1
}
```

**检测逻辑:**
1. 用 `document.querySelectorAll('form')` 或自定义容器
2. 对每个 form，用 `form.querySelectorAll('input, select, textarea, button, [contenteditable]')`
3. 对每个字段提取:
   - `label`: `form.querySelector('label[for="' + field.id + '"]')` 或用 `field.closest('label')`
   - 如果没 label，检查 `aria-label`, `aria-labelledby`, `placeholder`, `title`
   - `autocomplete`: 从 `<input autocomplete="...">` 获取
   - `options`: select 的 `<option>` 列表，radio 的同 name 列表
   - 最佳 selector: 优先 id → name → data-* → nth-child

#### 1.2 `fillForm(params)` — 升级版批量填充

```
功能: 按检测到的字段结构批量填充（取代现有的基础版本）
输入: 
  - formSelector: 目标 form（可不指定，自动找第一个）
  - fields: [{
      selector: "#email",    // 字段选择器
      value: "user@test.com",
      method: "write" | "select" | "check" | "uncheck" | "file"
    }]
  - submitAfter: true/false   // 填充后是否提交
  - delay: 200                // 字段间延迟（ms）
```

#### 1.3 `autoFill(params)` — 智能自动填充

```
功能: 自动检测字段，根据智能匹配注入数据
输入:
  - formSelector: 目标 form
  - profile: {
      // 按常见字段名称匹配的数据模板
      name: "张三",
      email: "zhangsan@test.com",
      phone: "13800138000",
      password: "...",
      address: "...",
      city: "北京",
      province: "北京市",
      zip: "100000",
      note: "备注内容",
      // 通用: 自定义键值对
      custom: { "company": "xxx公司" }
    }
  - strategy: "strict" | "fuzzy"   // 匹配策略
  - submitAfter: false
  - autoDetect: true                // 是否自动检测 form

智能匹配算法:
  1. 检测表单字段 → 获取字段元数据列表
  2. 对每个字段，按优先级匹配 profile 中的数据:
     a. label 匹配: label含"邮箱"/"email" → profile.email
     b. name 匹配: name="phone"/"tel"/"手机" → profile.phone
     c. placeholder 匹配: placeholder含"输入密码" → profile.password  
     d. id 匹配: id="address" → profile.address
     e. autocomplete 匹配: autocomplete="tel" → profile.phone
     f. aria-label/title 匹配
  3. 根据字段类型选择填充方法:
     - text/email/password/number → writeText (用原生 setter)
     - select → 按 text 或 value 选中 option
     - checkbox/radio → 按值勾选
     - textarea → writeText
     - file → 提示无法自动填充
  4. 返回填充结果报告

匹配关键词表（中英文）:
  name / 姓名 / 用户名 / 昵称 → profile.name
  email / 邮箱 / 邮件 → profile.email
  phone / 电话 / 手机 / 手机号 / tel / mobile → profile.phone
  password / 密码 → profile.password
  address / 地址 / 住址 → profile.address
  city / 城市 / 区 / 县 → profile.city
  province / 省 / 省份 / 地区 → profile.province
  zip / postcode / 邮编 / 邮政编码 → profile.zip
  note / comment / 备注 / 说明 / 留言 → profile.note
  company / 公司 / 单位 / 企业 → profile.custom.company
  search / 搜索 → (跳过, 不填搜索框)
  code / 验证码 / captcha → (跳过, 不填验证码)
```

#### 1.4 `validateForm(params)` — 表单验证

```
功能: 检查表单字段是否已正确填充
返回每个字段的填充状态
```

### 步骤 3: `hermes_client.py` — CLI 命令

新增 3 个命令:

```bash
# 检测表单字段
hermes_client.py detect_fields [formSelector]

# 批量填充
hermes_client.py fill_form <formSelector> '<json_fields>' [submitAfter]

# 智能自动填充
hermes_client.py auto_fill [formSelector] '<json_profile>' [strategy]
```

参数示例:
```bash
# 检测
python hermes_client.py detect_fields
python hermes_client.py detect_fields "#loginForm"

# 批量填充（profile 用 JSON）
python hermes_client.py auto_fill "" '{
  "name": "张三",
  "email": "test@test.com",
  "phone": "13800138000"
}'

# 指定填充策略
python hermes_client.py auto_fill "#registerForm" '{"name":"张三"}' "fuzzy"
```

## 六、风险和注意事项

| 风险 | 级别 | 对策 |
|------|:----:|------|
| Vue/React 响应式劫持 | 🟡 | 已有 `writeText` 的原生 setter，复用 |
| iframe 内表单 | 🟡 | 一期不支持，二期考虑 |
| Shadow DOM 表单 | 🟡 | 一期不支持 |
| 验证码输入框被误填 | 🟢 | autocomplete="off" 跳过 + 关键词"验证码/验证/captcha"跳过 |
| 动态渲染的表单 | 🟢 | 需先确保表单已渲染 |
| CSP 限制 eval | 🟢 | 全用 DOM API，无需 eval |
